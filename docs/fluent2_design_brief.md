## Fluent 2 设计说明（项目内简明稿）

角色定位（面向团队）

- 本项目属于“系统伴侣 / 控制中枢”类产品——碳足迹仪表盘与个人碳账本，适合采用 Fluent 2 家族视觉体系。Fluent 2 在此处的作用是：传达“现代、轻盈、有系统感”的视觉基底，让界面与 Windows 桌面产生自然的亲和力。

场景说明

- 适用场景：云服务控制台、效率工具、设置面板、桌面端配套 Web 控制台。用户进入界面后应立即有熟悉感：视觉上像“系统自带”的控制面板，而非独立的网页应用。

视觉理念

- 核心：柔和渐变背景 + 亚克力玻璃卡片 + 清晰轮廓与圆角系统。
- 背景：使用浅色渐变或柔和纹理（可借助 Unsplash 提供的软色抽象图像作为情感底图），叠加半透明渐变层以控制对比。
- 卡片：主体信息放在半透明的亚克力卡片上，使用 `backdrop-filter: blur()` + 高透明度白色叠层，边缘带细腻高光描边。
- 色彩：以 Fluent 蓝为主色，辅以蓝灰中性色；按钮与图标采用 Fluent 系统色系，配合简洁线性图标。

材质与动效

- 材质：强调“亚克力”质感——半透明、模糊、淡高光；避免过度噪点或强烈纹理。
- 动效：轻量空间位移（translateY -2~-6px）、柔和阴影与颜色过渡，时长 150–300ms，采用 Fluent 常用缓动曲线。

交互与可访问性

- 交互反馈以“微位移 + 渐变”为主，避免夸张弹跳。
- 强制检查对比度、焦点轮廓、键盘可达与 aria 标签，确保系统式产品的可用性。

实现建议（工程落地要点）

1. 使用 `@fluentui/web-components`（Web Components / 非 React 方案）或 `@fluentui/react-components`（React 项目）作为控件基础。
2. 在 `:root` 中定义主题 tokens（颜色/半透明度/圆角/阴影/动效时长），并在组件样式中引用。
3. 背景层使用 Unsplash 抽象/柔和图像并叠加渐变层，卡片使用 `backdrop-filter` 保持可读性。
4. 保留自定义可视化（TreeViz / Timeline），控件层使用 Fluent 组件以保持系统一致性。

总结

Fluent 2 为本项目提供“系统伴侣”级的视觉基底：让用户在 Web 控制台中感受到与 Windows 系统的视觉亲和，同时通过轻量化的动效与可访问性约束保持产品的专业性与易用性。
